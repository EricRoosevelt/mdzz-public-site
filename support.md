# MD摘摘 帮助文档

更新时间：2026-04-20

欢迎使用 MD摘摘。本文档面向首次安装、正在接入集成、或在回归测试阶段需要快速排障的用户。

## 1. 快速开始

### 1.1 这个扩展能做什么

MD摘摘 提供四类核心能力：

- 在支持站点中摘录对话内容到摘摘舱（Workspace）
- 从已保存内容回到原会话位置
- 将提示词注入当前聊天输入框
- 导出内容到 Word、Markdown、TXT，或推送到 Notion / Obsidian

### 1.2 基础使用路径

1. 打开任一支持站点的聊天页面。
2. 点击扩展图标打开 Sidepanel。
3. 在 `摘摘舱` 中查看摘录，在 `Prompts` 中管理提示词，在 `Settings` 中管理功能开关与接入配置。

## 2. 摘摘舱使用说明

### 2.1 摘录内容

你可以通过以下方式向摘摘舱添加摘录：

- 页面中的可见摘录入口
- 复制捕获
- 其他已接入的站点行为入口

摘录后的内容会带上：

- 来源站点
- 来源链接
- 时间戳
- 标签

### 2.2 标签与筛选

摘摘舱支持对摘录进行：

- 单条标签编辑
- 批量标签操作
- 标签筛选
- 搜索

建议：

- 用站点标签区分来源
- 用主题标签区分任务
- 用日期标签辅助回溯

### 2.3 原文定位

如果摘录保存了足够的来源信息，可以从摘摘舱触发“定位原文”。

若站点 DOM 已变化、消息已删除、或页面不受支持，扩展会尝试降级处理，并给出通知。

### 2.4 导出

当前支持三种导出格式：

- Word
- Markdown
- TXT

导出检查点：

- 文件名会做安全清洗
- 代码块与正文会尽量完整保留
- 导出失败会给出错误提示，不会静默失败

## 3. Notion 接入教程

### 3.1 接入前准备

你需要准备：

- 一个 Notion Internal Integration Token
- 一个目标 Database ID
- 一个可访问的 Notion API Endpoint

默认 Endpoint：

```text
https://api.notion.com/v1
```

### 3.2 配置步骤

1. 打开 `Settings > 接入 Notion`
2. 填写：
   - API Endpoint
   - Internal Integration Token
   - Database ID
3. 点击“测试连接”
4. 测试成功后再执行保存

### 3.3 常见错误

`Notion 鉴权失败`

- Token 不正确
- Token 已过期
- Integration 没有访问目标数据库的权限

`Notion 数据库校验失败`

- Database ID 错误
- 当前 Integration 未被共享到该数据库

`连接超时`

- 网络环境异常
- Notion API 请求被代理或防火墙阻断

## 4. Obsidian 接入教程

### 4.1 接入前准备

你需要准备：

- 本地可访问的 Obsidian 服务地址
- 如果你的服务启用了鉴权，再准备对应 API Key / Bearer Token

常见 Endpoint 示例：

```text
Vault Base URL: http://127.0.0.1:27123/vault/
Note Path Template: {{date}}/{{time}}-{{title}}.md
```

### 4.2 配置步骤

1. 打开 `Settings > 接入 Obsidian`
2. 填写：
   - Vault Base URL
   - Note Path Template
   - API Key / Bearer Token（如果启用了鉴权）
3. 点击“测试连接”
   - 如果你填写的是自定义域名或自托管地址，扩展会在首次测试时请求该地址的网络访问授权
   - 静态 `.md` 文件名已禁用；模板至少要包含一个变量，默认会避免覆盖已有笔记
4. 测试成功后执行保存

### 4.3 常见错误

`连接校验失败`

- Endpoint 写错
- 本地服务未启动
- 本地服务未允许当前来源访问

`目标文件尚不存在`

- 这是可继续推送的提示，不一定表示配置错误
- 首次推送时，服务应尝试创建目标 Markdown 文件

`扩展缺少目标域名权限，或目标服务拒绝请求`

- 目标地址不在允许的 host 范围内
- 本地服务 CORS 或鉴权策略拒绝了请求

## 5. 功能开关

在 `Settings > 功能开关` 中可以统一管理：

- 自动目录
- 划词捕获
- 复制捕获

说明：

- 关闭开关后，相关行为会真正停止，而不仅仅是 UI 变灰。
- 开关保存后，无需重装扩展即可生效。

当当前页面不属于支持站点时：

- Sidepanel 仍可打开
- 需要页面注入的能力可能不可用
- 扩展应给出可解释提示，而不是静默失败

## 7. 常见问题排查

### 7.1 页面没有反应

请先检查：

1. 当前是不是支持站点
2. 页面是否加载完成
3. 扩展是否已重新加载
4. 当前功能开关是否启用

### 7.2 当前版本如何完成摘录

首发版本统一通过页面内可见入口、复制摘录和侧边栏操作完成主要流程。

如果某个页面入口暂时不可用，请优先：

1. 确认当前页面属于支持站点
2. 检查 `划词捕获` 和 `复制捕获` 开关是否开启
3. 通过 Sidepanel 中的现有入口继续操作

### 7.3 导出失败

可能原因：

- 浏览器拦截了下载
- 页面缺少可用的下载容器
- 当前摘录内容异常

建议：

- 重试导出
- 检查浏览器下载权限
- 先改导出其他格式确认是否为单一格式问题

### 7.4 集成推送失败

优先检查：

- Endpoint 是否正确
- Token / Database ID 是否正确
- 本地服务或第三方接口是否在线

### 7.5 定位原文失败

常见原因：

- 原页面消息结构已变化
- 会话内容已被删除
- 站点 DOM 与适配器不再匹配

## 8. 隐私与权限摘要

当前扩展使用到的关键权限包括：

- `storage`：保存 snippets、prompts、设置与接入配置
- `sidePanel`：打开扩展侧边栏
- `activeTab`：基于当前活动页面执行注入、定位与上下文读取

扩展不会因为帮助文档的查看而额外申请新权限。
