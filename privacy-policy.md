# MD摘摘 隐私政策

更新时间：2026-04-20

> 本文档为扩展内与公开页面共用版本，用于 Chrome Web Store 上架与后续对外说明。

## 1. 我们是谁

MD摘摘 是一款 Chrome 扩展，用于帮助用户在支持的 AI 对话站点中摘录内容、管理提示词、导出文档并接入个人知识工具。

- 邮箱：ericroosevelttt@163.com
- 项目主页：https://ericroosevelt.github.io/mdzz-public-site/

## 2. 本政策覆盖什么

本政策说明：

- 扩展会读取哪些数据
- 数据为什么会被读取或保存
- 数据存放在哪里
- 数据是否会共享给第三方
- 用户如何控制和删除数据

## 3. 扩展会处理哪些数据

### 3.1 用户主动保存的内容

扩展可能会处理以下由用户主动触发保存的数据：

- 摘录内容
- 摘录来源链接与来源站点
- 标签与时间戳
- prompt 内容
- prompt 标签与预设状态

这些内容仅在用户执行摘录、编辑、导出或推送时被处理。

### 3.2 设置与接入配置

扩展可能会在本地存储以下配置：

- 自动目录开关
- 划词捕获开关
- 复制捕获开关
- 主题设置
- Notion 接入配置
- Obsidian 接入配置

### 3.3 技术性错误信息

当扩展发生错误时，可能记录必要的技术信息以便排查，例如：

- 错误代码
- 错误域
- 失败的链路上下文

扩展不会在生产环境中故意打印用户的长文本、私密 prompt、Token 或完整敏感正文到可见日志中。

## 4. 我们如何使用这些数据

这些数据只用于以下目的：

- 在摘摘舱中展示和管理摘录
- 在 Prompts 中展示和管理提示词
- 支持原文定位、提示词注入与导出
- 将用户主动选择的内容推送到 Notion 或 Obsidian
- 在发生错误时给出可解释反馈

我们不会将这些数据用于广告投放、画像分析或出售给第三方。

## 5. 数据存储在哪里

默认情况下，数据主要存储在用户本地浏览器的 `chrome.storage.local` 中。

这意味着：

- 数据默认保存在用户自己的浏览器环境里
- 扩展不会为了基础功能把内容上传到自建服务器

如果用户主动配置并使用了第三方集成：

- 推送到 Notion 的内容会发送到 Notion API
- 推送到 Obsidian 的内容会发送到用户提供的本地或自托管服务地址

## 6. 第三方服务

### 6.1 Notion

当用户主动配置并使用 Notion 集成时，扩展会向用户填写的 Notion API Endpoint 发起请求，以验证连接并创建页面内容。

### 6.2 Obsidian

当用户主动配置并使用 Obsidian 集成时，扩展会向用户填写的 Endpoint 发起请求，以写入 Markdown 内容。

对于这些第三方服务，数据将受其各自的隐私政策和安全机制约束。

## 7. 权限说明

扩展当前使用的主要权限包括：

### `storage`

用于保存：

- 摘录
- 提示词
- 功能开关
- 主题
- 第三方接入配置

### `sidePanel`

用于打开和展示扩展侧边栏界面。

### `activeTab`

用于在当前活动页面执行与用户当前操作直接相关的能力，例如：

- 提示词注入
- 原文定位
- 页面上下文读取

### 站点 host 权限

扩展只对已支持的目标站点申请对应 host 权限，以及对用户主动配置的 Notion / 本地服务接入地址申请所需访问范围。扩展不会使用 `<all_urls>`。

## 8. 数据共享与出售

我们不会出售用户数据。  
我们也不会把用户的摘录内容、提示词内容或设置上传到自建分析平台进行商业化处理。

只有在用户主动触发集成推送时，相关内容才会发送到用户指定的第三方服务。

## 9. 数据保留与删除

本地数据由用户自行控制。

用户可以通过以下方式移除数据：

- 在扩展内删除摘录或提示词
- 清空浏览器中该扩展的本地存储
- 卸载扩展

对于已推送到第三方服务的数据，需要用户在对应第三方服务中自行删除。
